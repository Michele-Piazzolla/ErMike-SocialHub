📚 Documentazione tecnica completa
ErMike-SocialHub

Analisi approfondita del social network realizzato con Node.js, Express, WebSocket, JSON e Ollama.

📌 1. Panoramica e stack tecnologico

ErMike-SocialHub è una piattaforma social sviluppata come progetto scolastico. Consente registrazione, login, pubblicazione di post con testo e immagini, like, commenti, follow/unfollow, profili pubblici, chat globale in tempo reale (WebSocket), messaggistica privata 1:1 e un assistente AI (@bot) basato su Ollama in locale.

ComponenteTecnologia
RuntimeNode.js (≥ v18)
Framework HTTPExpress 4.18
Template engineEJS
WebSocketws (libreria npm)
Upload fileMulter
Sessioniexpress-session
PersistenzaFile JSON locali (cartella data/)
AI BotOllama (modello llama3.2:1b)
FrontendHTML5/CSS3 puro + JavaScript

Il server ascolta sulla porta 3000 e gestisce contemporaneamente richieste HTTP e WebSocket.

🏗️ 2. Architettura del software

L'applicazione segue un pattern MVC semplificato:

  • Model – file JSON in data/
  • View – template EJS in views/
  • Controller – route in server.js

Struttura delle cartelle (rilevante):

├── server.js
├── views/
│   ├── login.ejs, register.ejs, home.ejs, createPost.ejs
│   ├── chat.ejs, privateChat.ejs, profile.ejs, profilePublic.ejs
│   └── partials/ (navbar.ejs, styles.ejs)
├── data/
│   ├── users.json, posts.json, comments.json
│   ├── chat.json, messages.json, follows.json
├── uploads/ (immagini post e avatar profilo)
├── public/ (file statici: CSS, immagini)
└── package.json

HTTP e WebSocket condividono la stessa porta. Il modulo ws intercetta le richieste di upgrade e mantiene una Map() dei client connessi.

🔌 5. WebSocket – protocollo e flussi

Il server centralizza tutte le connessioni in una Map(ws → username). I messaggi scambiati hanno un campo tipo numerico.

tipoDirezioneDescrizione
0client → serverAutenticazione: il client invia il proprio username. Il server lo registra e risponde con lo storico globale (tipo 1).
1server → clientStorico iniziale della chat globale (array di messaggi).
1client → serverNuovo messaggio globale. Il server lo salva su chat.json e lo diffonde a tutti (tipo 2).
2server → clientBroadcast di un nuovo messaggio globale a tutti i client connessi.
10client → serverInvio messaggio privato a un utente specifico (campi: to, text). Il server salva su messages.json, cerca il destinatario nella Map e glielo inoltra (tipo 10). Invia conferma anche al mittente.
10server → clientConsegna di un messaggio privato (sia al destinatario che al mittente per conferma).
-1server → clientErrore (es. username già connesso da un'altra scheda).

La chat privata supporta la consegna offline: se il destinatario non è connesso, il messaggio viene comunque salvato e sarà visibile quando aprirà di nuovo la pagina delle chat private (grazie al caricamento dello storico via HTTP).

⚙️ 6. Funzionalità implementate

🔐 Autenticazione e sessioni

Registrazione con username e password (memorizzate in chiaro – solo a scopo didattico). Login: verifica credenziali e crea una sessione server-side con express-session. Il middleware requireLogin() garantisce che tutte le route protette siano accessibili solo agli utenti autenticati.

📝 Post e feed

Creazione post: testo obbligatorio o immagine (almeno uno dei due). Le immagini vengono caricate con Multer e salvate in uploads/ con nome univoco. Il feed home mostra tutti i post ordinati per data decrescente. L'autore può eliminare i propri post (verifica server).

❤️ Like e commenti

Like: toggle senza ricaricare la pagina tramite fetch POST a /like/:postId. Il server restituisce JSON con il nuovo conteggio e lo stato liked. Il client aggiorna l'interfaccia. Commenti: sezione nascosta, caricata dinamicamente via GET /comments/:postId. Il form invia il commento via POST e la lista viene ricaricata in AJAX.

👥 Follow / Unfollow e profili pubblici

Ogni utente può seguire altri. La relazione è salvata in follows.json. Il profilo pubblico (es. /profile/mike) mostra avatar, bio, numero di post, follower, seguiti e presenta due bottoni: "Segui" (toggle) e "Messaggio" (collegamento alla chat privata). La password non viene mai inviata al client – viene costruito un oggetto safeUser.

🔍 Ricerca utenti (navbar)

Campo di ricerca con debounce (300 ms). La chiamata GET /search?q=... restituisce al massimo 6 utenti (escludendo l'utente corrente). I risultati sono mostrati in un dropdown con avatar e nome completo. Cliccando si accede al profilo pubblico.

💬 Chat globale

Pagina /chat. Connessione WebSocket autenticata, carica lo storico dei messaggi globali. I messaggi propri sono allineati a destra (bolla verde), quelli altrui a sinistra (grigio scuro). Per il bot AI (vedi sotto) le bolle hanno uno stile speciale (viola).

🤖 Chat privata 1:1

Pagina /private-chat con due colonne: a sinistra la lista delle conversazioni (utenti con cui si è già scambiato almeno un messaggio), a destra l'area di chat. Lo storico viene caricato via HTTP. L'invio usa WebSocket tipo 10. Se il destinatario è online riceve il messaggio in tempo reale, altrimenti il messaggio viene salvato e verrà visualizzato al successivo accesso alla chat.

🤖 AI Bot (Ollama)

Quando un utente scrive un messaggio nella chat globale che inizia con @bot, il server estrae la domanda, chiama il modello Ollama in locale (porta 11434) e invia la risposta con username AI Bot. Viene applicato un cooldown di 3 secondi per evitare sovraccarico. Il bot è completamente gratuito e privato, non richiede chiavi API.

// esempio di chiamata a Ollama
const response = await fetch('http://localhost:11434/api/generate', {
  method: 'POST',
  body: JSON.stringify({
    model: 'llama3.2:1b',
    prompt: `...`,
    stream: false
  })
});

🛡️ 7. Sicurezza e robustezza

  • Protezione route: requireLogin() reindirizza a /login se la sessione non è presente.
  • Controllo autore per eliminazione post: il server confronta l'username della sessione con post.username; se diverso restituisce 403.
  • Self-follow bloccato: se follower === followed il server risponde con 400 Bad Request.
  • Migrazione automatica: all'avvio il server aggiunge campi mancanti (bio, fullName, avatar, likes[]) agli utenti e post esistenti, garantendo la compatibilità con versioni precedenti.
  • SafeUser nei profili pubblici: la password non è inclusa nell'oggetto passato alla view.
  • Gestione errori WebSocket: ogni messaggio JSON è avvolto in un try/catch; errori di parsing non causano il crash del server.
  • Creazione preventiva delle cartelle: uploads/profiles/ viene creata all'avvio per evitare errori ENOENT durante l'upload dell'avatar.

🧪 8. Test eseguiti e limitazioni note

Test manuali effettuati con successo:

  • Registrazione, login, logout
  • Creazione post con solo testo, solo immagine, entrambi – e post vuoto bloccato
  • Like e unlike su post propri e altrui – contatore aggiornato
  • Commenti: apertura sezione, invio, visualizzazione
  • Eliminazione post (solo autore) – rimozione dal DOM e dal JSON
  • Follow/unfollow – aggiornamento contatori e bottone
  • Profilo pubblico e modifica profilo personale
  • Ricerca utenti e navigazione al profilo
  • Chat globale su due browser – messaggi in tempo reale
  • Chat privata online/offline – consegna immediata o posticipo
  • AI Bot – risposta a @bot, cooldown, stile diverso

Limitazioni attuali:

  • Password salvate in chiaro (in produzione usare bcrypt).
  • Nessun limite alla dimensione dei file upload (Multer senza fileSize limit).
  • Lo storage JSON non è thread‑safe e non adatto a più processi Node.js (per demo va bene).
  • Il campo read nei messaggi privati è salvato ma non ancora utilizzato nell'interfaccia (manca l'indicatore di "non letto").
  • Il feed home non filtra per utenti seguiti (mostra tutti i post). La logica di filtro è predisposta ma non attivata.

▶️ 9. Avvio del progetto

Requisiti: Node.js ≥ 18, npm, (opzionale) Ollama per il bot.

git clone <repository>
cd ErMike-SocialHub
npm install express@4.18.2 express-session ejs multer ws
node server.js

Il server sarà accessibile su http://localhost:3000.

Per l'AI Bot: installare Ollama (ollama.com), avviare il servizio e scaricare un modello:

ollama pull llama3.2:1b

Assicurarsi che Ollama sia in esecuzione prima di chiamare @bot.

Nota sulle versioni: Usare Express 4.x (non 5.x) per evitare incompatibilità con path-to-regexp.

🚀 10. Possibili sviluppi futuri

  • Feed personalizzato: mostrare solo i post degli utenti seguiti (già predisposto nel codice, basta attivare il filtro).
  • Notifiche in tempo reale (like, commenti, nuovi messaggi) via WebSocket.
  • Password hashate con bcrypt.
  • Indicatore di "letto/non letto" per i messaggi privati.
  • Chat di gruppo.
  • Supporto per allegati diversi dalle immagini (PDF, video).
  • Integrazione con un database relazionale (SQLite, PostgreSQL).
