📡 3. Distance Vector – esempio di “count to infinity”

In una rete a 3 router (A–B–C), se il collegamento B–C si interrompe, l'algoritmo Distance Vector può impiegare molti passi per convergere (problema count to infinity). Si usa split horizon o route poisoning per mitigarlo.

Situazione iniziale: A→B=1, B→C=1, A→C via B costo 2.
Se B–C cade, B crede ancora che C sia raggiungibile tramite A (costo 3) → loop.

📚 Teoria completa del routing (apri i menù per studiare)

📌 1. Concetti base: cos'è il routing ▼

Il routing è il processo di selezione del percorso che i pacchetti devono seguire per attraversare una rete IP da un'origine a una destinazione. Avviene a livello 3 (rete) del modello ISO/OSI. I dispositivi che svolgono il routing sono i router.

Ogni router mantiene una tabella di routing che contiene le rotte verso le reti conosciute. Le rotte possono essere configurate staticamente o apprese dinamicamente tramite protocolli.

⚙️ 2. Routing statico vs dinamico ▼

Routing statico: l'amministratore inserisce manualmente le rotte. Vantaggi: nessun overhead, sicuro, adatto a reti piccole o stub. Svantaggi: non scalabile, non si adatta ai guasti.

Routing dinamico: i router scambiano informazioni tramite protocolli (RIP, OSPF, EIGRP, BGP) e aggiornano automaticamente le tabelle. Convergenza automatica in caso di cambiamenti.

Classificazione: IGP (Interior Gateway Protocol) – dentro un AS (RIP, OSPF) e EGP (Exterior Gateway Protocol) – tra AS (BGP).

🗂️ 3. Tabella di routing (struttura dettagliata) ▼

Ogni voce della tabella contiene:

  • Rete destinazione (indirizzo IP di rete)
  • Subnet mask (per identificare la porzione di rete)
  • Next hop (indirizzo IP del router successivo)
  • Interfaccia di uscita (eth0, wlan0, ...)
  • Metrica / costo (numero di hop, banda, ritardo, affidabilità)
  • Tipologia (statica/dinamica, default)

Il router esegue il longest prefix match: sceglie la voce con la subnet mask più specifica che corrisponde all'indirizzo di destinazione.

Comando per vedere la tabella su router Cisco: show ip route; su Linux: ip route show.

📊 4. Metriche di routing ▼

Le metriche sono valori usati per valutare la "bontà" di un percorso. Più bassa è la metrica, migliore è la rotta. Esempi:

  • Hop count: numero di router attraversati (usato da RIP, max 15).
  • Banda passante: privilegia percorsi con maggiore velocità (OSPF).
  • Ritardo (delay): tempo stimato di transito (EIGRP).
  • Carico (load): congestione del collegamento.
  • Affidabilità (reliability): tasso di errore del collegamento.
  • Costo amministrativo (AD): preferenza tra protocolli (es. AD statico=1, OSPF=110, RIP=120).

Metriche composte (EIGRP) combinano banda + ritardo.

🔁 5. Algoritmi di routing: Distance Vector (Bellman-Ford) ▼

Distance Vector (DV): ogni router condivide con i vicini la propria tabella (vettore delle distanze). Aggiorna le rotte usando l'equazione di Bellman-Ford: d(x,y) = min_v ( c(x,v) + d(v,y) ).

Protocolli DV: RIP (Routing Information Protocol) – hop count, max 15, invia aggiornamenti ogni 30 secondi. Problemi: convergenza lenta, count-to-infinity.

Soluzioni: split horizon (non inviare una rotta al router da cui è stata appresa), poisoned reverse (invia la rotta con metrica infinita), timers hold-down.

🧠 6. Algoritmi Link State (Dijkstra) ▼

Link State (LS): ogni router costruisce una mappa completa della rete (LSDB – Link State Database) scambiando LSA (Link State Advertisement). Usa l'algoritmo di Dijkstra per calcolare il percorso minimo.

Fasi:

  1. Scoperta dei vicini (hello packet).
  2. Costruzione LSA (costo dei collegamenti).
  3. Flooding delle LSA a tutta l'area.
  4. Esecuzione Dijkstra su LSDB.

OSPF (Open Shortest Path First) è il protocollo LS più diffuso. Vantaggi: convergenza rapida, nessun loop, supporto subnet mask (VLSM). Svantaggi: maggiore consumo di CPU/memoria.

// Pseudocodice Dijkstra (semplificato)
dist[src] = 0
per ogni nodo v: dist[v] = infinito
coda priorità = {src}
mentre coda non vuota:
  u = nodo con distanza minima
  per ogni vicino v di u:
    alt = dist[u] + costo(u,v)
    se alt < dist[v]: dist[v] = alt, padre[v]=u
🌍 7. Routing tra domini (BGP) ▼

BGP (Border Gateway Protocol) è il protocollo EGP usato su Internet. Basato su percorsi (path vector). Non usa metriche tradizionali ma attributi (AS_PATH, NEXT_HOP, LOCAL_PREF, MED).

BGP impedisce loop grazie all'AS_PATH (lista di AS attraversati). È un protocollo lento ma molto scalabile, usato dai provider.

🛠️ 8. Comandi e configurazione (esempi Linux/Cisco) ▼

Linux (iproute2):
ip route add 192.168.2.0/24 via 10.0.0.1 dev eth0 (statica)
ip route del 192.168.2.0/24
ip route show
Per routing dinamico: usare quagga o frr.

Cisco IOS:
ip route 192.168.2.0 255.255.255.0 10.0.0.1
router rip / router ospf 1 / network 10.0.0.0
